EqualTo Sheets Beta

Welcome to the EqualTo Sheets beta! This document outlines some of the key features of the platform.

What is EqualTo Sheets?

EqualTo Sheets is a “Spreadsheets as a service” platform for developers. We want EqualTo Sheets to be the quickest, most convenient way to add spreadsheet capabilities to your website or product.

In summary, you can do the following in the EqualTo Sheets beta:

  • Create a blank workbook.

  • Upload an existing XLSX workbook.

  • Edit a workbook using the EqualTo Sheets widget.

  • Create/read/update/delete workbook sheets and cells using our REST and GraphQL APIs

  • Simulate “what if” scenarios using our REST API (“If cell Sheet1!A1 is changed to 60, what value is calculated in Sheet1!D20?”).

Later in this document, we’ll explain in detail how to accomplish the above.

What is the purpose of the beta?

We think EqualTo Sheets is something new: developer-first spreadsheet technology. With the beta, we aim to learn from our users where EqualTo Sheets can bring the most value, so that we can plan our product development accordingly.

If you see potential in what we have, get in touch: diarmuid.glynn@equalto.com! We want to talk and learn more about how we can help you accomplish your goals.

GraphQL API

The “Authorization” header will be automatically set, so that you can immediately start querying EqualTo Sheets.

Note that the GraphiQL client also contains some auto-generated documentation on the API:

Some sample queries:

List all workbooks

query {
	workbooks {
		id
		name
	}
}

Create a blank workbook:

mutation {
	createWorkbook {
		workbook {
			id
			name
		}
	}
}

List the sheets in a specific workbook:

query {
	workbook(workbookId: "<workbook id>") {
		id
		name
		sheets {
			name
		}
	}
}

View the contents of A1 in the first sheet of a workbook:

query {
   workbook(workbookId: "<workbook id>") {
   	id
   	name
   	sheet(sheetId: 1) {
   		cell(ref: "A1") {
   			value {
   				boolean
   				text
   				number
   			}
   			formattedValue
   		}
   	}
   }
}

Set cell A1 in the first sheet of a workbook to “300$”:

mutation {
	setCellInput(workbookId:"<workbook id>", sheetId: 1, row: 1, col: 1, input: "300$") {
		__typename
	}
}

Simulation API

You must set the HTTP “Authorization” header to “Bearer <license key>” to use the REST APIs.

The Simulation API allows you simulate the impact of modifying data in a workbook.

GET or POST

/api/v1/workbooks/<workbook id>/simulate?inputs=<inputs>&outputs=<outputs>

workbook id: GUID of the relevant workbook

inputs:      JSON dictionary, {<sheet names>: {<cell refs>: <value>}}
             example: {"Sheet1": {"A1": 500}}

outputs:     JSON dictionary, {<sheet name>: [<cell refs>]}
             example: {"Sheet1": ["B2", "B3"]}

For example, this will respond with the value that Sheet1!F10 would contain if Sheet1!C6 was set to 50000:

curl 'https://sheets.equalto.com/api/v1/workbooks/<workbook id>/simulate?inputs=%7B%22Sheet1%22%3A%7B%22C6%22%3A50000%7D%7D&outputs=%7B%22Sheet1%22%3A%5B%22F10%22%5D%7D' -H 'Authorization: Bearer <license key>'

How does authentication work?

You should have already received a “license key”, which is used to access your resources on sheets.equalto.com. Your license key is sensitive, and you should keep it secret. Anyone with your license key can access all your EqualTo Sheets data.

The license key is used to securely access resources on sheets.equalto.com via Bearer Authentication. The license key is inserted into the HTTP “Authorization” header: Authorization: Bearer <license key>

In production use, you should proxy requests to sheets.equalto.com, so that you can securely insert the required bearer authentication without revealing the license key on the client:

If you are simply privately testing EqualTo Sheets, then you can skip the proxy server and use the license key in your client code. But ensure you don’t deploy code to production that works in that manner:

What Excel features do you support?

We support:

  • 179 Excel functions (out of ~450)

  • Cell styling and formatting

Partial support (we’re working on it):

  • Merged cells

  • Defined names

  • Excel Tables

We don’t support (TODO):

  • Pivot tables

  • Conditional formatting

  • Dynamic array functions, such as FILTER()

  • Various date/time functions, such as TODAY()

  • Arrays and array formulas

  • Borders

  • Charts

We have no plans to support:

  • VBA

  • Power Query
